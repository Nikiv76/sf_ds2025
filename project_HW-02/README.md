**Дополнительные условия:**

Необходимо реализовать вывод графической информации построенной с помощью библиотеки Plotly в репозитории GitHub в Preview.
- Библиотека Plotly позволяет строить интерактивные графики, которые можно приближать, отдалять, а также просматривать значения на графике в реальном времени. К тому же в библиотеке собрано огромнейшее количество красочных методов визуализации. У Plotly приятный дизайн, а способов работы с ней несколько.
- С помощью Plotly можно делать сложные визуализации с элементами управления, например строить интерактивную 3D-визуализацию, карту мира и многое другое.

Построенные с помощью библиотеки Plotly графики, диаграммы и др. графики отображаются в Jupyter Notebook, а в репозитории GitHub в Preview без дополнительного кода эти графики не видны.
Реализация вывода графиков Plotly в Preview репозитория GitHub будет осуществлена двумя способами:
1. Вставка графической информации (графика), предварительно сохраненного в формате PNG, в Markdown Cell (Ячейка Markdown). Реализацию можно будет посмотреть в файле churn_analys.ipynb.
2. При помощи библиотеки Kaleido, которая позволяет Plotly экспортировать графики в статические форматы (PNG, JPEG, PDF), но с дополнительным кодом. Реализацию можно будет посмотреть в файле churn_analys2.ipynb.

### Краткая информация о данных.

Информация представлена в табличной форме в формате CSV, со следующим списком названий столбцов (признаков):
* RowNumber — номер строки таблицы (это лишняя информация, поэтому можете сразу от неё избавиться)
* CustomerId — идентификатор клиента
* Surname — фамилия клиента
* CreditScore — кредитный рейтинг клиента (чем он выше, тем больше клиент брал кредитов и возвращал их)
* Geography — страна клиента (банк международный)
* Gender — пол клиента
* Age — возраст клиента
* Tenure — сколько лет клиент пользуется услугами банка
* Balance — баланс на счетах клиента в банке
* NumOfProducts — количество услуг банка, которые приобрёл клиент
* HasCrCard — есть ли у клиента кредитная карта (1 — да, 0 — нет)
* IsActiveMember — есть ли у клиента статус активного клиента банка (1 — да, 0 — нет)
* EstimatedSalary — предполагаемая заработная плата клиента
* Exited — статус лояльности (1 — ушедший клиент, 0 — лояльный клиент)ю

 Форма таблицы:
  * строк - 10000
  * столбцов - 22

Размер документа составляет 1,1 МБ
